[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the architecture diagram to see how it works under the hood. (pause)
First, every query goes through our Security Checkpoint node to scrub PII and filter dangerous requests before it reaches the main AI. (pause)
Then, the clean query is received by the orchestrator_agent, which coordinates the workflow and delegates tasks. (pause)
For scheduling and records, it delegates to the vet_scheduler_agent, which checks clinic slots. (pause)
For food recommendations and compatibility, it delegates to the diet_advisor_agent. (pause)
These sub-agents call domain-specific tools hosted on a stdio-based MCP Server, such as search_vets, check_food_safety, and log_pet_task. (pause)
Finally, for vet scheduling, the orchestrator routes the request to a human_approval_node to pause and request user confirmation before logging the appointment. (pause)
